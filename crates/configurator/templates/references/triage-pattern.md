# Issue / PR 分类模式（轻量）

用于对 issue 和 PR 进行分类和分派。适合维护开源项目或管理任务队列的项目。

## 角色

**分类**（categories）：
- `bug` — 某东西坏了
- `enhancement` — 新功能或改进

**状态**（states）：
- `needs-triage` — 未评估
- `needs-info` — 等待 reporter 补充
- `ready-for-agent` — 规范完善，agent 可实现
- `ready-for-human` — 需人工实现
- `wontfix` — 不处理

## 流程

1. **收集上下文** — 读取 issue/PR 全文（body、comments、labels、author、dates；PR 还需 diff）
2. **推荐** — 告知分类和状态的推荐理由，等待指示
3. **验证** — 如果是 bug，从 reporter 步骤复现；如果是 PR，确认 diff 符合描述
4. **详细化**（如需） — 用 `dj-grill` 方法逐个问题细化
5. **应用结果** — 标注分类+状态，关闭或加注释

## 硬规则

- Ai 生成的注释以 `*This was generated by AI during triage.*` 开头
- 每个 issue 承载一个分类 + 一个状态
- 不重复问已解决的问题

## 适用条件

此模式需要项目有可访问的 issue tracker（GitHub Issues 等）。DiJiang 本身没有内置 issue tracker。
