# 决策阶梯（Decision Ladder）

在写每一行代码前，逐层自检。通过一层就不进下一层。

```
1. YAGNI ── 这行代码真的需要吗？
     ↓ 需要
2. 复用现有 ── 项目里已经有现成的了吗？
     ↓ 没有
3. 标准库 ── 语言 stdlib 能解决吗？
     ↓ 不能
4. 原生特性 ── 语言本身的特性够用吗？
     ↓ 不够
5. 已有依赖 ── 已有依赖提供需要的能力吗？
     ↓ 不提供
6. 一行代码 ── 一行能搞定吗？
     ↓ 不能
7. 最小代码 ── 写最少能工作的代码
```

## 规则

- 每一层检查通过就停下，不进下一层
- "复用现有" 包含项目内已有工具函数、常量、类型
- "标准库" 包括语言内置模块和全局 API
- "已有依赖" 指 `package.json` / `go.mod` / `Cargo.toml` 等中已声明的第三方库
- "一行代码" 是字面意义上的一行，不含链式调用
- 跳过任意层需要注释说明原因（如"stdlib 版本有安全漏洞，使用 X 替代"）
