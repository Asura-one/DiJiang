# 强度等级（Intensity Levels）

某些 `dj-*` skill 支持强度等级，控制分析的深度和输出范围。

## 等级定义

| 等级 | 特征 | 适用场景 |
|------|------|----------|
| **lite** | 最小检查/输出，只覆盖核心路径 | 日常快速审查、小改动的代码检查 |
| **full** | 完整检查，覆盖所有维度（默认） | 标准 PR/功能交付 |
| **ultra** | 深度检查，包括边界情况、性能、替代方案 | 关键模块、上线前最终检查、新人加入 |

## 默认值

除非用户明确指定，所有 skill 使用 `full` 等级。

## 声明方式

用户可以在请求时指定等级：

```
/dj-check lite        # 仅做核心路径验证
/dj-check             # 默认 full
/dj-check ultra       # 深度全量检查
```

## 支持强度等级的 skill

| Skill | lite | full | ultra |
|-------|------|------|-------|
| dj-ponytail | 跳过安全底线外的所有非必要代码 | 遵守所有规则 | 每行都有贡献证明 |
| dj-check | 仅类型检查 + 核心测试 | 功能+安全+回归 | 全量 + 性能 + 边界 |
| dj-review | 仅 spec 匹配度 | spec + 代码质量 | 双 agent + 替代方案审查 |
| dj-audit | 仅检查可见的过度工程 | 过度工程 + 安全 | 全量 + 每个文件行数检查 |

## 实现原则

- 低等级向上兼容：lite 做的事 full 也做
- 等级只影响检查/分析深度，不影响正确性要求
- 安全底线在任意等级都不能跳过
