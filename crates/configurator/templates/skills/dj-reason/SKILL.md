# Reason: 推理增强

用结构化思考分析复杂判断和架构取舍。不是替代决策，而是让决策过程显式化。

## 认知矩阵

分析方案时，从这些维度展开：

| 维度 | 问题 |
|---|---|
| 目标 | 这个方案要解决什么问题？ |
| 约束 | 哪些是不可变的硬约束？ |
| 取舍 | 选了这个，放弃了什么？ |
| 风险 | 什么情况下这个方案会失败？ |
| 成本 | 实现和维护的成本估计？ |

## 系统透镜

从系统角度分析影响：

1. **局部 vs 全局** — 这个改动对当前模块好，但对整个系统呢？
2. **短期 vs 长期** — 现在快但以后慢？还是现在慢但以后快？
3. **可逆 vs 不可逆** — 错了能回滚吗？数据迁移、API 发布是不可逆的

## 流程

1. 陈述要分析的决策
2. 列出候选方案（至少 2 个）
3. 用认知矩阵分析每个方案
4. 用系统透镜评估影响面
5. 给出推荐和理由

## 边界

- 不做执行——只分析、推荐、报告
- 不给"完美方案"——所有方案都有取舍
- 如果两个方案都很差，说实话
