# 吸收拆解清单 (Absorption Checklist)

用于 `dj-absorb` 的 Phase 1。与 `dj-remix` 的拆解清单不同——这里不追求全量功能覆盖，而是从"什么值得吸收"的视角审视目标。根据吸收范围和目标类型选择对应维度。

## 通用筛选维度

### UI 组件模式

- [ ] 布局方案（网格系统、间距体系、响应式策略）
- [ ] 卡片/列表/表格的样式设计（圆角、阴影、间距比例）
- [ ] 表单组件设计（输入框样式、校验反馈方式）
- [ ] 按钮系统（层级体系、尺寸系统、图标搭配）
- [ ] 导航组件（侧边栏、顶部导航、标签页样式）
- [ ] 弹窗/抽屉/浮层设计（层级、动画、遮罩）
- [ ] 加载状态设计（骨架屏样式、spinner风格、进度条）
- [ ] 空状态 / 报错状态的视觉设计
- [ ] 通知/Toast 系统（位置、动画、类型配色）
- [ ] 头像/徽章/标签等微小组件

### 交互模式

- [ ] 页面切换过渡动画
- [ ] 列表/卡片进场动画
- [ ] 表单交互（实时校验、自动保存、输入联动）
- [ ] 搜索/筛选交互方式（即时搜索、条件组合、标签筛选）
- [ ] 拖拽/排序/重排交互
- [ ] 无限滚动 / 分页加载方式
- [ ] 手势操作（滑动、长按、缩放——移动端）
- [ ] 悬浮/右键菜单
- [ ] 快捷键体系
- [ ] 微交互细节（按钮点击反馈、开关切换动画、数字变化动效）

### 视觉语言

- [ ] 配色系统（主色、辅助色、背景色、文字色、边框色——完整调色板）
- [ ] 字体搭配（标题字体、正文字体、等宽字体、字号层级）
- [ ] 间距体系（统一的间距比例/基准）
- [ ] 圆角体系（各层级组件的圆角统一性）
- [ ] 阴影/层级体系（不同z-depth的阴影定义）
- [ ] 图标风格（线性/面性、圆角风格、粗细统一性）
- [ ] 插画风格（如果有）
- [ ] 深色/浅色模式的转换设计
- [ ] 动效语言（缓动函数、持续时间、运动轨迹风格）

### 数据模型与架构

- [ ] 数据结构设计（实体关系、字段组织、嵌套层级）
- [ ] API 设计模式（RESTful 风格、GraphQL 查询结构、实时通信方式）
- [ ] 路由设计模式（扁平/嵌套、权限控制、懒加载策略）
- [ ] 状态管理模式（全局状态/局部状态的划分策略）
- [ ] 缓存策略（数据预取、缓存失效、乐观更新）
- [ ] 错误处理模式（全局错误边界、错误展示方式、重试策略）
- [ ] 日志/监控的埋点方式

### 用户体验细节

- [ ] 首次加载/新手引导设计
- [ ] 表单填写体验（自动填充、分步表单、保存草稿）
- [ ] 搜索体验（搜索建议、搜索历史、空结果处理）
- [ ] 批量操作方式（全选、批量编辑、批量删除）
- [ ] 比较/对比功能（如果有）
- [ ] 导出/分享功能设计
- [ ] 消息/通知系统（是否已读、分组、推送）
- [ ] 帮助/文档内嵌方式（工具提示、帮助中心、引导气泡）

### 信息架构

- [ ] 导航层级深度与宽度
- [ ] 内容分类方式
- [ ] 页面间跳转关系
- [ ] 搜索/发现内容的组织方式
- [ ] 内容详情页的信息层级和阅读顺序

## 候选元素卡片模板

每个候选元素记录以下信息：

```
## 候选元素卡片

来源：[目标名称]
元素类型：[UI 组件 / 交互模式 / 视觉语言 / 数据模型 / 架构模式 / 微交互]
描述：[简短描述这个元素]
吸収价值：[为什么值得吸收？比自有项目现有哪些优势？]
适配成本：[融入自有项目需要多大改动？]
冲突风险：[与自有项目现有风格/架构的冲突程度：高/中/低]
是否吸收：[是 / 否 / 待讨论]
理由：[判断理由]
依赖条件：[该元素在源项目中依赖什么条件才能工作？自有项目是否具备？]
替代方案：[如果不吸收这个，有没有其他方式达到类似效果？]
```

## 筛选判断原则

| 条件 | 判断 | 动作 |
|------|------|------|
| 吸収价值高 + 适配成本低 + 冲突风险低 | ✅ 强烈推荐 | 直接吸收 |
| 吸収价值高 + 适配成本中/高 + 冲突风险中/高 | 🤔 待讨论 | 进入 Phase 2 出融合方案后判断 |
| 吸収价值低 + 任何适配成本/冲突风险 | ❌ 不吸收 | 标记为"淘汰"并记录理由 |
| 同类元素来自多个目标 | 🔍 择优 | 比较后选择最有价值的，或设计统一方案 |
| 技术栈不兼容 | ❌ 不吸收 | 记录"不适合"并说明技术原因 |
| 依赖的条件在自有项目中不存在 | ⚠️ 有条件吸收 | 需先评估引入依赖条件的成本 |
