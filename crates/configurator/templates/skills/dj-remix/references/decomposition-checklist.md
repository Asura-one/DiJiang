# 拆解清单参考 (Decomposition Checklist)

用于 SKILL.md 的 Phase 1。动手列清单前先扫一遍这份参考，避免漏项。分两部分：**通用清单**（任何网站/App都适用）和 **按类型的专项补充**（根据复刻对象的类型挑对应部分看）。

## 通用清单

### 信息架构
- [ ] 完整的页面/路由列表（含二级、三级页面，不只是顶级导航能看到的）
- [ ] 导航层级（主导航、面包屑、侧边栏、footer链接）
- [ ] 登录前 vs 登录后可见的页面差异
- [ ] 权限/角色差异（如果有多种用户角色，各角色能看到的页面是否不同）

### 页面骨架（对每个页面都要过一遍）
- [ ] 从上到下的区块顺序
- [ ] 每个区块的布局形式（网格/列表/卡片/轮播）
- [ ] 页面顶部/底部的固定元素（sticky header、悬浮按钮）

### 组件与状态
- [ ] 按钮：default / hover / active / disabled / loading
- [ ] 表单输入框：empty / filled / focus / error / disabled
- [ ] 卡片/列表项：default / hover / selected
- [ ] 弹窗/抽屉/下拉菜单：打开/关闭动画、点击外部关闭
- [ ] 通知/Toast：成功/失败/警告样式
- [ ] 加载状态：骨架屏、spinner、进度条
- [ ] 空状态：列表为空、搜索无结果时展示什么
- [ ] 报错状态：表单校验错误、页面级错误（404/500）

### 交互与行为
- [ ] 表单校验规则（必填、格式、实时校验 vs 提交时校验）
- [ ] 搜索/筛选/排序的具体逻辑
- [ ] 分页 vs 无限滚动
- [ ] 拖拽排序/上传
- [ ] 快捷键
- [ ] 动效/过渡（页面切换、元素进出场）

### 响应式
- [ ] 移动端/平板/桌面端的断点
- [ ] 移动端导航形式（汉堡菜单/底部tab）是否和桌面端不同
- [ ] 触屏手势（滑动、长按）

### 全局元素
- [ ] Header/Footer/主导航
- [ ] 主题模式（深色/浅色/跟随系统）
- [ ] 多语言切换（如果有）
- [ ] Cookie/隐私提示条
- [ ] 客服/反馈入口

### 容易漏掉的细节
- [ ] Tooltip/帮助文案
- [ ] 面包屑
- [ ] 徽章/标签（新功能标记、数量角标）
- [ ] 版权信息/法律条款链接
- [ ] 无障碍细节（alt文字、键盘可达性、对比度）

## 按类型的专项补充

### 电商类
- 商品列表页的筛选维度、排序方式
- 商品详情页：多图切换、规格选择、库存状态、评价区、推荐商品
- 购物车：数量调整、优惠券、运费计算
- 结算流程：地址、支付方式、订单确认
- 订单管理：订单状态流转、物流跟踪

### SaaS / 后台管理类
- 仪表盘：图表类型、数据筛选维度、自定义视图
- 表格：排序、筛选、批量操作、列自定义、导出
- 设置页：账号、团队成员/权限管理、账单/订阅
- 通知中心

### 内容 / 博客 / 资讯类
- 文章列表：分类、标签、摘要展示形式
- 文章详情：目录导航、评论区、相关推荐、分享按钮
- 搜索结果页

### 社交 / 社区类
- 动态流：点赞/评论/转发交互
- 个人主页：关注/粉丝、内容tab切换
- 消息/通知系统
- 实时更新机制（新内容提示）

### 营销 / 落地页类
- Hero区域的CTA设计
- 定价表
- 客户案例/评价墙
- 表单转化点（留资表单、订阅框）

### 移动 App（如果复刻对象是App）
- 底部Tab导航 vs 侧滑菜单
- 手势交互（下拉刷新、滑动删除）
- 原生控件的呈现方式（弹出层、选择器）
- 状态栏/安全区域适配
