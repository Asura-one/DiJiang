# 工具偏好与外部知识

> **目的**：统一工具和包管理器选择、搜索策略以及外部知识检索实践。

---

## 包管理

| 技术栈 | 管理器 | 说明 |
|-------|---------|-------|
| Python | `uv` | 始终使用 uv 管理 venv、依赖和脚本 |
| JavaScript / TypeScript | `pnpm` | 通过 lock 文件确认（pnpm-lock.yaml vs package-lock.json vs yarn.lock） |
| 开发工具 | `mise` | 管理 Bun、Node.js、pnpm 版本 |

## 代码搜索

使用可用的搜索工具（grep/rg 或 agent 原生搜索），按以下顺序：

1. **精确匹配优先** — 已知函数名、类名、常量 → 精确搜索
2. **模糊搜索兜底** — 位置不确定、需要语义理解 → 放宽搜索模式
3. **穷举引用点** — 找到定义后，搜索所有引用点确认影响范围
4. **逐层深入** — 先粗后细：先定位文件，再定位函数，再定位行

### 禁止行为

- 不读代码就猜测代码位置
- 凭框架惯例直接回答
- 不读目标代码就提出修改建议

## 外部知识检索

需要当前信息时才搜索，已知稳定事实不搜。

- 简单事实：搜索 1 次
- 中等任务：搜索 3-5 次
- 深度研究：搜索 5-10 次
- 优先使用内部来源（代码库、文档），再使用外部网络搜索
- 结果冲突时继续搜索，直到找到明确答案
- 不确定的信息标注"待确认"

### 引用与版权

- 用自己的话复述，不直接复制粘贴
- 直接引用：≤15 个词，每个来源最多引用一次
- 不确定来源时，宁可不引用，也不要编造
