# {{project_name}} Coding Guidelines

## Rust Version

This project targets Rust {{rust_version}} (MSRV).

## Code Style

- Follow `rustfmt` conventions.
- Clippy must pass with no warnings.
- Prefer `thiserror` for error types and `anyhow` for error propagation.

## Architecture (Depth Module Principle)

Modules should be deep: a small interface with substantial behavior behind it.

| Principle | Rule |
|-----------|------|
| Single Responsibility | One module = one clear job |
| Minimal Interface | Expose the minimum necessary surface |
| Position | Put code where it belongs, don't create new files casually |

## Testing

- Unit tests live alongside the code they test (`#[cfg(test)] mod tests`).
- Integration tests live in `tests/`.
- All public functions must have at least one test.

## Commit Messages

Use conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`.

## Dependency Management

1. Check stdlib first.
2. Check existing workspace deps.
3. Write the minimum code to solve the problem.
